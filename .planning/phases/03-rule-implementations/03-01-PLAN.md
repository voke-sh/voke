<objective>
Implement the 8 Schema Correctness rules (MTQS-S01 through MTQS-S08) as pure synchronous
`RuleFunction`s, packaged as a single `schemaRules: RuleDefinition[]` export in its own module.

Purpose: Schema Correctness is the T1 (1.5x) correctness floor — a broken inputSchema means the
tool literally cannot be used. These 8 rules cover RULE-01 in full.
Output: `packages/linter/src/rules/schema.ts` exporting `schemaRules`, plus a test file with
positive + negative fixtures for every rule, all running under a network-block stub (RULE-06).

This plan owns ONLY files under `packages/linter/src/rules/schema.ts` and `tests/`. It does NOT
touch `registry.ts` or `index.ts` — wiring is done by the Wave 2 integration plan (03-05) to
avoid parallel file contention.
</objective>
